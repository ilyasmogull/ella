Html Files & Structure

All HTML codes are valid. There are no HTML errors. Mostly there are 3 main parts in our HTML pages. Header + Main + Footer section.

Sections are placed in the page wrapper class including Notification Bar, Header, Main, Footer, Pop-up and Sidebar.

Demos & Folder Structures

All home pages( index-*.html pages) are easy to separate and create a new demos. Whole images that are used in these assets/pages are placed in the assets/images/banners/home folder. (home-1, home-2 etc...)

All additional css/sass/js code used in these demos are also separated and placed in their separate folders. css / assets - sass / assets - js We will talk more about these in other sections.

SASS(Scss) / Css / Js Files

Main Css file of Ella is style.css(These are also generated with Sass). Ella is based of bootstrap framework so bootstrap.min.css file is also important one.

Sass folder have extra css files(These are also generated with Sass) that has styles for index-1 to index-10 pages and pages. These contains extra codes for those demos.

Main Js file of Ella is theme.js. You can change or create new function from this folder

lib folder has styles for the plugins that we used in Ella.

You can make changes on Sass file or you can directly edit css files. That's your choice. Both ways are easy to use and code is well commented to guide you.

skins folder has the skins.scss/skins.css files for demos. For example index-2.html page uses skin-2.css file. You can change or create new skins from this folder or you can see the Sass section to change it via variables.
